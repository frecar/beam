Quick Start

Install on Ubuntu 24.04

Add the APT repository and install. GPU recommended but not required.

One-liner Recommended 
curl -fsSL https://frecar.github.io/beam/install | sudo bash
Manual APT setup 
# Add the signing key
curl -fsSL https://frecar.github.io/beam/gpg/beam.gpg \
  | sudo tee /etc/apt/keyrings/beam.gpg > /dev/null

# Add the repository
echo "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/beam.gpg] \
  https://frecar.github.io/beam/apt stable main" \
  | sudo tee /etc/apt/sources.list.d/beam.list

# Install
sudo apt-get update && sudo apt-get install beam

After installation, open https://your-server:8444 in any browser and log in with a Linux account. See the README for build-from-source instructions and configuration.

Why Beam

Features

Everything you need to stream a full Ubuntu desktop to any browser.

GPU

GPU-Accelerated Encoding

NVIDIA NVENC, AMD/Intel VA-API, or x264 software fallback. Auto-detected at startup.

<30

Ultra-Low Latency

Sub-30ms round-trip on LAN via WebRTC. Hardware-decoded directly in the browser.

www

Zero-Install Client

Works in Chrome, Firefox, Safari. No plugins, no app downloads. Just open a URL.

PAM

Multi-User Sessions

Isolated virtual desktops per user with PAM authentication and JWT session tokens.

~

Audio and Clipboard

PulseAudio streaming with Opus encoding. Bidirectional clipboard sync between local and remote.

1:1

Dynamic Resolution

Desktop resolution automatically matches your browser viewport. Adaptive bitrate adjusts to network conditions.

||

Session Persistence

Sessions survive server restarts. Reconnect without re-login after connection loss.

kb

Keyboard and Input

Auto keyboard layout detection, Mac Cmd-to-Ctrl remapping, cursor shape passthrough, smooth trackpad scrolling.

Under the Hood

How It Works

Three components: a browser client, a signaling server, and a per-user desktop agent.

Browser

  • WebRTC video/audio
  • Keyboard and mouse
  • Clipboard sync
  • Local cursor rendering
←→ HTTPS / WS

Server

  • PAM auth + JWT
  • WebSocket signaling
  • Session management
  • Agent lifecycle
←→ spawn

Agent

  • XCB/SHM capture
  • GStreamer encode
  • WebRTC SRTP
  • uinput injection
