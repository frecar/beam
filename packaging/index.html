Quick Start

Install Beam

Requires Ubuntu 24.04+ or Debian 13+. GPU recommended but not required — falls back to software encoding.

One-liner Recommended 
curl -fsSL https://raw.githubusercontent.com/frecar/beam/gh-pages/install | sudo bash
Manual APT setup 
# Add the signing key
curl -fsSL https://raw.githubusercontent.com/frecar/beam/gh-pages/gpg/beam.gpg \
  | gpg --dearmor | sudo tee /etc/apt/keyrings/beam.gpg > /dev/null

# Add the repository
echo "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/beam.gpg] \
  https://raw.githubusercontent.com/frecar/beam/gh-pages stable main" \
  | sudo tee /etc/apt/sources.list.d/beam.list

# Install
sudo apt-get update && sudo apt-get install beam

After installation

  1. Beam starts automatically as a systemd service
  2. Open https://your-server:8444 in Chrome or Firefox
  3. Accept the self-signed certificate warning (first time only)
  4. Log in with any Linux user account on the machine
  5. Run beam-doctor if anything isn't working

See the README for build-from-source instructions, TLS configuration, and advanced setup.

FAQ

Common questions

Quick answers to the most common questions about Beam.

What GPU do I need?
NVIDIA GPUs with NVENC (Maxwell generation or newer, roughly GTX 900+) give the best performance. AMD and Intel GPUs work via VA-API. No GPU at all? Beam falls back to x264 software encoding — it just uses more CPU.
Does it work over the internet or just LAN?
Both. On LAN you'll see sub-30ms latency. Over the internet, latency depends on your connection — Beam's adaptive bitrate adjusts automatically. Beam uses a single HTTPS/WebSocket port (default 8444) — just ensure it's reachable from the client.
Which Linux distributions are supported?
The .deb packages are built and tested on Ubuntu 24.04. Any distribution with glibc 2.38+ and GStreamer 1.24+ should work — this includes Ubuntu 24.04+, Debian 13 (Trixie), and derivatives. Older versions (Ubuntu 22.04, Debian 12) are not compatible.
Is it secure?
Yes. All traffic is encrypted via TLS (HTTPS + WebSocket). Authentication uses Linux PAM with JWT session tokens and rate limiting. Each user session runs in an isolated X display. The code is open source and auditable.
Can I use it for gaming?
Beam is optimized for desktop and productivity workloads. It supports 60fps H.264 streaming with GPU encoding, but does not currently include controller passthrough or other gaming-specific features.
How does authentication work?
Beam uses Linux PAM — the same system that handles your login screen and SSH. Any local Linux user account can log in. Sessions are managed with JWT tokens that refresh automatically.

Under the Hood

How It Works

Three components: a browser client, a signaling server, and a per-user desktop agent.

Browser

  • WebCodecs video/audio
  • Keyboard & mouse input
  • Clipboard sync
  • Local cursor rendering
←→ HTTPS / WS

Server

  • PAM auth + JWT
  • Binary frame relay
  • Session management
  • Agent lifecycle
←→ spawn

Agent

  • XCB/SHM capture
  • GStreamer encode
  • WebSocket stream
  • XTEST injection
